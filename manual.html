사용자 매뉴얼

이 문서는 Microsoft 365 관리 센터 화면을 재현한 교육용 시뮬레이터의 전체 기능을 설명합니다. 실제 서비스와 연결되어 있지 않으므로 모든 조작을 자유롭게 연습할 수 있습니다.

개요

이 시뮬레이터는 학교 관리자가 Microsoft 365 관리 센터에서 수행하는 주요 업무 흐름을 실제 화면과 최대한 동일하게 연습할 수 있도록 만들어졌습니다. 다음 업무를 다룹니다.

분류기능
사용자 관리사용자 추가(단일/여러 명·CSV), 삭제, 암호 초기화, 라이선스 관리, 그룹 관리
계정 세부 정보이름 클릭 → 계정 편집: 이메일/별칭, 로그아웃, 연락처 정보, 관리자 역할, 그룹 멤버십, 라이선스 및 앱, OneDrive
보안사용자 단위 다단계 인증(MFA) — Entra 관리 센터 화면
Teams 및 그룹활성 팀 및 그룹 목록(3개 탭), 팀 추가, Microsoft 365 그룹 추가, 이름·설명 편집
SharePointOneDrive 보존/저장 한도 설정 — SharePoint 관리 센터 화면
✔ 모든 기능은 실제로 동작합니다. 추가한 계정은 목록에 나타나고, 삭제하면 사라지며, 라이선스 변경은 목록의 라이선스 열에 바로 반영됩니다.

데이터와 동작 원리

  • 학생 계정 100명이 자동 생성되어 있습니다. 표시 이름은 [2학년 5반]김민준 형식이고, 계정은 아이디@schoolname.sen.hs.kr입니다. 고정 시드 난수를 사용하므로 새로고침해도 같은 명단이 유지됩니다.
  • 라이선스는 약 75%가 학생용 Microsoft 365 A3 사용 혜택, 나머지는 Microsoft Power Automate Free가 함께 할당되어 있습니다.
  • 페이지를 새로고침하면 추가/삭제/변경한 내용은 초기화됩니다. (MFA 상태만 브라우저에 저장되어 유지)
  • 사이드바 메뉴는 기본적으로 모두 접혀 있으며, 화살표를 눌러 펼칠 수 있습니다.

라이선스 및 앱 · OneDrive 탭

  • 라이선스 및 앱 — 위치 선택(한국)과 라이선스 5종이 표시되고, 학생이 현재 보유한 라이선스가 미리 체크되어 있습니다. 체크를 바꾸면 변경 내용 저장이 활성화되고, 저장 시 활성 사용자 목록의 라이선스 열에 즉시 반영됩니다. 앱(n) 헤더를 펼치면 선택된 라이선스의 포함 앱을 볼 수 있습니다.
  • OneDrive — 파일 액세스 권한 가져오기, 사용한 저장소(0% of 5120 GB), 공유, 조직의 OneDrive 설정(데이터 보존 3650일 / 저장 공간 5120GB)이 표시됩니다.
    • 데이터 보존 관리 → SharePoint 관리 센터(sharepoint.html)가 새 탭으로 열리며 보존 패널(삭제된 사용자의 OneDrive 보존 일 수, 30~3650일)이 자동으로 열립니다.
    • 기본 저장소 관리 → 같은 페이지가 저장 한도 패널(기본 저장 한도 GB)과 함께 열립니다.
    • SharePoint 관리 센터에서도 설정 표의 OneDrive “보존”/“저장 한도” 행을 클릭해 패널을 열 수 있고, 사이드바 토글은 모두 닫힌 상태가 기본입니다.
ⓘ 패널을 닫았다가 다시 열면 항상 계정 탭부터 시작합니다. 장치·메일 탭은 표시용 안내만 제공합니다.

4사용자 추가 (4단계 마법사)

사용자 추가 버튼 | 홈 도구 모음 · 사용자 관리 카드 · 활성 사용자 도구 모음 어디서든

우측에서 큰 패널이 열리며, 좌측 진행 표시기(기본 사항 → 제품 라이선스 → 설정 → 마침)를 따라 진행합니다.

  1. 기본 사항 설정 — 성/이름을 입력하면 표시 이름이 자동 완성됩니다. 필수 항목은 표시 이름과 사용자 이름(도메인 @schoolname.sen.hs.kr 고정)입니다.
    • 자동으로 암호 만들기를 체크하면 12자리 암호가 자동 생성되고 입력란이 잠깁니다.
    • 암호 입력란의 눈 아이콘으로 표시/숨김 전환이 가능합니다.
    • 필수 항목이 모두 채워져야 다음이 활성화됩니다.
  2. 제품 라이선스 할당 — 위치(한국)와 라이선스를 선택합니다.
    • 라이선스 5종: Copilot Studio 바이럴 평가판, Power Apps for Developer, Power Automate Free, 교직원용 A3, 학생용 A3
    • 하단 앱(0) 헤더를 클릭하면 라이선스에 포함된 43개 앱 목록이 펼쳐집니다. 모두 선택 체크박스로 일괄 제어할 수 있고, Microsoft 365 Lighthouse(플랜 1)는 조직 수준 할당이라 개별 해제할 수 없습니다.
    • "제품 라이선스 없이 사용자 만들기(권장하지 않음)"를 고르면 라이선스 없이 진행됩니다.
  3. 설정(선택 사항) — 역할과 프로필 정보를 지정합니다.
    • 기본값은 사용자(관리자 센터 액세스 권한 없음)입니다. 관리 센터 액세스를 선택하면 AI 관리자, Exchange 관리자, 전역 관리자 등 역할 체크박스 10개가 활성화됩니다.
    • 프로필 정보: 직함, 부서, 사무실, 전화, 주소, 우편 번호, 국가 등을 입력할 수 있습니다.
  4. 검토 후 완료 — 입력한 모든 정보(표시 이름·계정, 암호 유형, 위치·라이선스·앱, 역할, 프로필)가 요약되며, 각 항목의 편집 링크로 해당 단계로 돌아갈 수 있습니다. 추가 완료를 누르면 계정이 활성 사용자 목록에 정렬 위치에 맞게 추가됩니다.
⚠ 어느 단계에서든 취소를 누르면 "닫으시겠습니까?" 확인 팝업이 뜹니다. 예를 누르면 입력한 모든 정보가 사라지고, 아니요를 누르면 작성이 유지됩니다.

5여러 사용자 추가 (CSV 일괄 등록)

사용자 › 활성 사용자 › 여러 사용자 추가

3단계 마법사(기본 사항 → 제품 라이선스 → 마침)로 여러 명을 한 번에 추가합니다.

방법 A. 그리드에 직접 입력

  • 표 형태 입력 그리드: 이름 · 성 · 표시 이름* · 사용자 이름* · 도메인
  • + 행 추가로 행을 추가하고, 행 끝 ✕로 행 제거, 선택 열(이름/성) 머리글의 ✕로 열 제거가 가능합니다.

방법 B. CSV 파일 업로드

  1. "사용자 정보가 포함된 CSV 파일 업로드" 헤더를 클릭해 섹션을 펼칩니다.
  2. 서식 파일을 내려받습니다.
    링크파일내용
    머리글만 포함된 CSV사용자_추가_서식.csv실제 M365 서식과 동일한 16개 열 머리글
    머리글 및 샘플 포함 CSV사용자_추가_샘플.csv머리글 + 예시 3명
  3. 엑셀에서 명단을 채운 뒤 찾아보기로 업로드하면 그리드에 자동으로 채워집니다.
ⓘ CSV 규칙 — 1열은 계정(사용자 이름, hong123@schoolname.sen.hs.kr처럼 도메인 포함 가능)이고 이후 이름, 성, 표시 이름 순입니다. 표시 이름이 비어 있으면 성+이름으로 자동 생성되고, 직함·부서 등 5열 이후는 무시됩니다. 머리글 행과 계정이 빈 행은 건너뜁니다.

2단계에서 선택한 위치·라이선스는 추가되는 전원에게 동일하게 적용되며, 마침 단계에서 인원·라이선스를 확인 후 추가 완료를 누르면 전원이 목록에 추가됩니다.

방법흐름
행 선택 없이 클릭사용자 선택 패널이 먼저 열립니다 — 검색, 모두 선택, 아바타+이름+계정 목록에서 대상 선택 → 선택
행을 체크한 뒤 클릭선택 패널을 건너뛰고 바로 확인 화면으로 이동

확인 화면 (인원에 따라 다름)

  • 1명 — "[표시 이름] 삭제" 상세 화면: 30일 복원 안내, 해제될 라이선스 배너, 전자 메일 및 OneDrive 옵션 4개(3개는 비활성, "다른 사용자에게 전자 메일 액세스 권한 부여"만 선택 가능)
  • 여러 명 — "사용자 삭제 / 사용자 N명 선택됨" 일괄 확인 화면과 30일 안내문

완료 화면

  • 1명 — "○○○ 님이 삭제되었습니다." + 라이선스 할당 취소 목록 + 계정 삭제 확인
  • 여러 명 — "✔ 삭제된 사용자: N" + 사용자/상태 표(각 계정에 User deleted)
⚠ 삭제를 실행하면 계정이 활성 사용자 목록에서 실제로 제거됩니다. (새로고침 시 초기 명단으로 복원)

7암호 초기화

행 호버 시 열쇠 아이콘 또는 행 선택 후 암호 초기화
  1. 우측 패널 상단에서 대상 계정을 확인합니다. 여러 명 선택 시 "○○○ 외 N명"으로 표시됩니다.
  2. 자동으로 암호 만들기(기본 체크)를 유지하거나, 해제하고 직접 입력합니다. "처음 로그인할 때 암호를 변경하도록 설정" 여부도 선택합니다.
  3. 암호 초기화를 누르면 초록 체크와 함께 12자리 임시 암호가 표시됩니다. 학생에게 안전한 방법으로 전달하세요.
✔ 초기화된 계정은 목록에서 열쇠 아이콘이 고정 표시되어 구분할 수 있습니다.

8제품 라이선스 관리

행 선택 › 제품 라이선스 관리
  1. "사용자의 라이선스를 어떻게 하시겠습니까?"에서 방식을 고릅니다.
    방식동작
    바꾸기기존 라이선스를 모두 해제하고 선택한 라이선스로 교체
    더 할당하기기존 라이선스를 유지하고 선택한 라이선스를 추가(중복 제외)
    모두 할당 해제모든 라이선스 제거 — 추가 선택 없이 바로 저장 가능
  2. 바꾸기/더 할당하기를 고르면 라이선스(n) 섹션이 나타납니다. 5종 중 하나 이상 체크해야 저장할 수 있습니다.
  3. 앱(n) 헤더를 펼치면 선택한 라이선스에 포함된 앱이 표시됩니다. "앱 표시" 드롭다운으로 라이선스별 필터링, 모두 선택으로 일괄 제어가 가능합니다. (예: Copilot Studio 바이럴 평가판 → CCI Bots용 Common Data Service 등 3개)
  4. 변경 내용 저장 → 저장 중 스피너(약 1초) → 결과 화면: "사용자 N/N명의 라이선스를 할당했습니다.", 할당된 라이선스 표(사용자 이름/최종 라이선스), ↓ 다운로드(결과 CSV 저장), 다음 단계 안내 링크.
✔ 변경 결과는 활성 사용자 목록의 라이선스 열에 즉시 반영됩니다.

9그룹 관리

행 선택 › ⋯ 메뉴 › 그룹 관리
  1. 패널 상단에서 선택 인원("사용자 N명 선택됨")과 안내 배너 "그룹 1개만 선택할 수 있습니다."를 확인합니다.
  2. 그룹 멤버십 목록에서 그룹을 고릅니다. 검색창에서 그룹 이름/이메일로 필터링할 수 있고, 한 그룹을 체크하면 행이 하늘색으로 강조됩니다. 다른 그룹을 체크하면 기존 선택이 자동 해제됩니다(1개 제한).
  3. 변경 내용 저장 → 스피너(약 1초) → 목록 화면으로 돌아오며 초록 배너 "✔ 변경했습니다."가 표시됩니다. 체크는 풀리고 방금 저장한 그룹 행의 강조만 유지됩니다.
ⓘ 목록에는 활성 팀 및 그룹의 팀/그룹이 그대로 나오므로, 팀 추가로 만든 그룹도 즉시 선택할 수 있습니다.

10다단계 인증(MFA)

활성 사용자 도구 모음 › 다단계 인증 › 새 탭으로 Entra 관리 센터 열림

검정 헤더의 Microsoft Entra 관리 센터 화면("사용자 단위 다단계 인증")이 별도 탭으로 열립니다. 학생 명단은 관리 센터와 자동으로 공유됩니다.

사용 방법

  1. 표(이름 | UPN | 상태)에서 대상 학생을 체크합니다. 검색창과 "필터 초기화"도 사용할 수 있습니다.
  2. 상태에 따라 상단 작업 버튼이 활성화됩니다.
    버튼활성 조건동작
    MFA 사용disabled 사용자 선택 시"다단계 인증 활성화" 대화 상자(aka.ms/mfasetup 링크) → 사용 → 상태 enabled
    MFA 사용 안 함enabled/enforced 선택 시상태를 disabled로 되돌림
    MFA 적용enabled 선택 시상태를 enforced로 변경
    사용자 MFA 설정1명 이상 선택 시우측 패널: 연락 방법 재제공 / 앱 암호 삭제 / 디바이스 MFA 복원 체크 후 저장
  3. 작업이 완료되면 우측 상단에 초록 체크 토스트("다단계 인증이 활성화됨" 등)가 뜨고 종 아이콘에 알림 배지가 붙습니다.
✔ MFA 상태는 브라우저(localStorage)에 저장되어 탭을 닫거나 새로고침해도 유지됩니다.

12팀 / Microsoft 365 그룹 추가 (5단계 마법사)

활성 팀 및 그룹 › 팀 추가 또는 Microsoft 365 그룹 추가

전체 페이지 마법사(기본 사항 → 소유자 → 구성원 → 설정 → 마침)입니다. 두 버튼의 흐름은 같고 문구만 팀/그룹으로 달라집니다.

  1. 기본 사항 — 팀 이름(그룹은 "이름")은 필수, 설명은 선택입니다.
  2. 소유자 추가 — 입력란에 이름/이메일을 치면 기존 계정 자동 완성 목록이 뜨고, 선택하면 아바타 칩으로 추가됩니다(✕로 제거). 소유자가 없으면 빨간 안내 "소유자를 입력하세요."와 함께 다음으로 진행할 수 없습니다.
  3. 구성원 추가 — 같은 방식으로 추가하되 최대 20명까지입니다. 20명이 되면 입력이 잠기고 "구성원은 최대 20명까지 추가할 수 있습니다."가 표시되며, 칩을 제거하면 다시 입력할 수 있습니다. 구성원은 비워도 됩니다.
  4. 설정 편집 — 팀 전자 메일 주소(필수, 접미사 @schoolname.sen.hs.kr 고정)를 입력하고 공개 범위(공개/비공개)를 드롭다운에서 선택합니다. 비어 있으면 빨간 테두리와 안내가 표시됩니다.
  5. 검토 및 완료 — 기본 사항/소유자/구성원/설정 요약과 항목별 편집 링크. 팀 추가(그룹은 "그룹 추가")를 누르면 목록에 즉시 추가됩니다. 그룹은 팀 상태 아이콘 없이 표시됩니다.
⚠ 취소를 누르면 "닫으시겠습니까? — 팀(그룹)에 대해 입력한 모든 정보가 손실됩니다." 팝업이 뜹니다.
